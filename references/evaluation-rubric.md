# 5次元評価ルーブリック

各イテレーションで以下の5次元を10点満点で評価する。

---

## 1. デザイン品質（閾値: 7/10）

### 検証方法
- `preview_screenshot` でデスクトップ（1280x800）とモバイル（375x812）を撮影
- `preview_inspect` で以下のCSSプロパティを測定

### 評価項目

| 項目 | 基準 | 検証 |
|------|------|------|
| ブランドカラー一貫性 | primary/accent が全ページで統一 | inspect: color, background-color |
| セクションpadding | >= 80px（py-20相当） | inspect: padding-top, padding-bottom |
| コンテンツ幅 | max-width: 1080px、中央配置 | inspect: max-width, margin |
| カード角丸 | 12px（rounded-xl） | inspect: border-radius |
| 背景色リズム | セクションごとに白/グレー/ダーク交互 | screenshot目視 |
| タイポグラフィ | フォント統一、ウェイト適切 | inspect: font-family, font-weight |
| 画像エリア | プレースホルダーが適切に配置 | screenshot目視 |
| ホワイトスペース | 詰まりすぎ・空きすぎがない | screenshot目視 |

### スコアガイド
- 9-10: プロのデザイナーが作ったように見える
- 7-8: 企業サイトとして十分な品質
- 5-6: テンプレート感は残るが破綻なし
- 3-4: 明らかなデザイン不備がある
- 1-2: 素人が作った感が強い

## 2. コピーライティング（閾値: 7/10）

### 検証方法
- 生成された `src/data/*.ts` と各ページのテキストをAIが自己評価
- ターゲット顧客の視点でレビュー

### 核心基準
**「ターゲット顧客が読んで、次のアクションを取りたくなるか？」**

### 共通評価項目（全業種）

| 項目 | OK | NG |
|------|-----|-----|
| 語り口 | 顧客の言葉で書かれている | 業界用語だらけ |
| 具体性 | 何をしてくれるか明確 | 「ソリューションを提供」等の曖昧表現 |
| 共感性 | 顧客の悩み・欲求に寄り添っている | 自社の凄さの羅列 |
| 行動喚起 | 次のアクションが明確 | CTAが曖昧 |
| ポエム回避 | 地に足のついた表現 | 「想いをつなぐ」「未来を創造」 |
| 差別化 | その会社ならではの強み | どの会社にも当てはまる汎用文 |

### 業種別の追加基準

| 業種 | 追加で重視するポイント |
|------|-------------------|
| consulting | 専門性の証明、伴走感、実績数字 |
| medical | 安心感、症状の分かりやすい説明、予約の簡単さ |
| restaurant | 食材・調理へのこだわり、雰囲気の伝達、予約導線 |
| it-service | 課題→解決のストーリー、導入事例のROI、料金の明快さ |
| manufacturing | 品質管理体制、設備・技術力、納期の信頼性 |
| construction | 施工事例のビフォーアフター、費用の透明性、保証 |
| beauty | 施術メニューの分かりやすさ、スタッフの雰囲気、気軽さ |
| education | 受講者の変化・成果、カリキュラムの具体性、講師の信頼性 |

### スコアガイド
- 9-10: ターゲット顧客が即座に行動を起こすレベル
- 7-8: ターゲット顧客が共感し、行動を検討するレベル
- 5-6: 情報は伝わるが響かない
- 3-4: 汎用テンプレートのまま
- 1-2: 意味不明・矛盾がある

## 3. 技術品質（閾値: 7/10）

### 検証方法
- `npm run build` でエラーゼロ
- `preview_inspect` でタイポグラフィ階層測定
- `preview_resize` でレスポンシブ確認

### 評価項目

| 項目 | 基準 | 検証 |
|------|------|------|
| ビルド | エラーゼロ | npm run build |
| タイポ階層 | H1 > H2 > H3 > p（単調減少） | inspect: font-size |
| line-height | 本文 >= 1.8 | inspect: line-height |
| レスポンシブ375px | 1カラム、崩れなし | screenshot mobile |
| レスポンシブ768px | 適切なカラム数 | resize + screenshot |
| レスポンシブ1440px | コンテンツ中央、余白十分 | screenshot desktop |
| セマンティクス | 適切なHTML要素使用 | snapshot |
| 構造化データ | schema.org JSON-LD 配置 | コードレビュー |

### スコアガイド
- 9-10: Lighthouse 90+相当
- 7-8: 主要な技術基準をすべて満たす
- 5-6: ビルドは通るが細部に問題
- 3-4: レスポンシブが崩れる
- 1-2: ビルドが通らない

---

## 4. アクセシビリティ（閾値: 6/10）

### 検証方法
- `preview_snapshot` でアクセシビリティツリー確認

### 評価項目

| 項目 | 基準 |
|------|------|
| 画像alt属性 | 全 `<img>` にalt指定 |
| 見出し階層 | h1→h2→h3 が順序通り（飛ばさない） |
| フォームラベル | 全input/textareaにlabel紐付け |
| リンクテキスト | 「こちら」ではなく意味のあるテキスト |
| コントラスト | 背景と文字の十分なコントラスト |
| キーボード操作 | フォーカス可視、Tab順序適切 |
| ランドマーク | header/main/footer/nav 使用 |

### スコアガイド
- 9-10: WCAG AA準拠
- 7-8: 主要なアクセシビリティ基準を満たす
- 5-6: 基本的な対応（alt, label）はある
- 3-4: 多数の問題あり
- 1-2: アクセシビリティ未考慮

---

## 5. コンバージョン（閾値: 6/10）

### 検証方法
- screenshot で CTA配置確認
- ページ構成からの導線分析

### 評価項目

| 項目 | 基準 |
|------|------|
| ヘッダーCTA | ヘッダーに問い合わせ/相談ボタン |
| Hero CTA | ファーストビューにCTAボタン |
| ページ末CTA | 全主要ページの末尾にCTAセクション |
| 問い合わせ到達 | トップから2クリック以内で問い合わせ |
| 信頼シグナル | 実績・資格・パートナー等の表示 |
| 悩み共感 | ターゲットの悩みに言及 |
| 差別化 | 競合との違いが明確 |

### スコアガイド
- 9-10: プロのマーケターが設計したレベル
- 7-8: コンバージョン導線が明確
- 5-6: CTAはあるが導線が弱い
- 3-4: CTAが不十分
- 1-2: コンバージョン設計なし
